# Bifrost v1.0.0

Bifrost reaches 1.0 with **Solutions** — installable, deploy-owned, lifecycle-aware packages of apps, workflows, forms, agents, tables, and configs that can be shipped between environments, installed from a zip or a GitHub repo, developed locally, and updated in place. This release also hardens the execution sandbox, formalizes the external-user access model, and moves the project to the `gobifrost` GitHub org and container registry. 652 commits since v0.9.1.

## ⚠️ Release cadence going forward

Starting with this release, Bifrost ships on a three-rung ladder:

- **`:dev`** — built on every merge to `main`. **Bleeding edge; expect bugs.** This is where the maintainer flushes out defects in his own production before they reach anyone else.
- **`vX.Y.Z-rc.N` (pre-release)** — cut **roughly monthly, in between full releases**. Intended to be **safer than `:dev`**: a versioned, signed candidate that's been through the gates and is being soak-tested, but not yet blessed. Pre-releases do **not** move the `:latest` tag.
- **`vX.Y.Z` (full release)** — **roughly monthly**, blessed/stable, tagged `:latest`. The default for production installs.

If you want fresher-than-monthly without riding `:dev`, pin to the latest `-rc` tag.

## Features

- **Solutions** — package a set of platform entities (apps, workflows, forms, agents, tables, configs, event/schedule triggers) into an installable unit. Install from a workspace zip (`bifrost solution install <zip>`) or from a GitHub repo (including an omni-repo subpath); develop locally with `bifrost solution start` (one-origin, Firebase/SWA-style — app dev server + in-process workflows, deploys nothing); ship with `bifrost solution deploy`.
- **Read-only, deploy-owned installs** — a Solution's entities are read-only in the target environment (editing one live returns 409). Updates flow through `deploy`, or through auto-pull for git-connected installs. An always-on database guard backs this up.
- **Connection references + Setup wizard** — a Solution declares the integrations it needs; install surfaces unmet connection needs, blocks/flags them, and a Setup wizard + checklist + README tab guide the operator to completion.
- **Export & backup** — `bifrost solution export --mode shareable` (code + schema, no secrets) or `--mode full` (password-protected, Fernet-encrypted secrets + data) for a portable, restorable backup.
- **Update signal & versioning** — installs compare descriptor versions to surface an update-available signal; git-connected installs auto-pull; redeploy applies a newer workspace.
- **SolutionDetail UI** — redesigned to three tabs (Overview / Contents / Configuration), with Setup demoted to an actionable banner+badge state that clears when setup completes.
- **`everyone` access tier** — a new access level that grants signed-in external/portal users; complements roles and the now-narrower `authenticated` tier.

## Security & Supply-Chain Hardening

- **Execution sandbox hardening** — worker pod security context (capability drop, seccomp, read-only root filesystem) and **credential-free execution children** so workflow code can no longer inherit secrets that could mint tokens or decrypt integration data; test-stack brought to parity (#363, #364).
- **External-user isolation closures** — external users no longer see global-tier configs, knowledge, or cross-org MCP agents; an e2e test proves zero global tools / zero global KB docs / no cross-org agent for external callers.
- Removed an orphaned `GET /api/integrations/sdk/{name}` endpoint that could leak any tenant's decrypted integration secrets to any authenticated user — use the org-scoped `POST /api/sdk/integrations/get`.
- Removed the developer-context org-override mechanism (a cross-org data-access vector); pass `scope` explicitly per SDK call.
- Routine base-image and GitHub Action bumps (node, nginx, python, codeql-action, codecov-action) via Dependabot.

## Reliability & Developer Experience

- **CLI contract gate** — the CLI now negotiates a `contract_version` with the server and hard-blocks on a mismatch, so an outdated CLI fails loudly instead of corrupting data.
- **Unified `--org` standard** — uniform org targeting across commands: `--org <id|name|none|global>` (with `--organization`/`--scope` as synonyms) plus a `--global` flag, with distinct home/global/org semantics.
- **Pre-release `-rc.N` release rung** wired end-to-end in CI (auto-detected pre-release, no `:latest`).
- Migrated to the `gobifrost` GitHub org; images now publish to `ghcr.io/gobifrost/*`.
- Process-pool worker reaping fix (#243 by @MTG-Thomas) and WebAuthn passkey-transport normalization (#357 by @MTG-Thomas).
- Reaped a large amount of dead code and unused dependencies across the SDK/CLI and execution paths.

## Breaking Changes

- **Removed `bifrost export` / `bifrost import`.** Superseded by Solutions. Old invocations now error (unknown command, exit 1) — no silent no-op. **Migration:** package with `bifrost solution deploy` / `solution export` and distribute with `bifrost solution install <zip>`; for raw `_repo/` workspace movement, use git sync.
- **App creation defaults to the v2 app model.** A bare `bifrost apps create <name>` (or `POST /api/applications` with no `app_model`) now returns **409**, directing you to scaffold/deploy v2 apps inside a Solution (`bifrost solution scaffold-app` + `solution deploy`). **Migration:** pass `--app-model inline_v1` (CLI) / `"app_model": "inline_v1"` (REST) to create a legacy standalone app.
- **CLI contract version advanced (now 4).** Older CLIs are blocked against the new server. **Migration:** upgrade your `bifrost` CLI to this release's build from `<API_URL>/api/cli/download`.
- **`access_level="authenticated"` now means "everyone except external users."** Inert on upgrade (all existing users default to non-external). **Migration:** when you start flagging users external, move entities they must reach to the new `everyone` tier or grant an explicit role.
- **`GET /api/integrations/sdk/{name}` removed** (security fix; zero in-repo callers). Migrate any third-party script to `POST /api/sdk/integrations/get`.
- **`developer_contexts` removed** (security fix); per-user override rows are dropped on upgrade. Pass `scope` explicitly per SDK call.
- Custom claims can now be global (`organization_id` widened to nullable) — additive, no action required.

## Solutions v1.0.0 — coverage caveats

Knowledge/RAG corpora, form/table file uploads, and private-repo install are **not** bundled/supported yet. Agents reference knowledge by namespace name, and the namespace must already exist in the target environment.
